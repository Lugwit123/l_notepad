已经继续改了，方向已经从 UI 彻底往数据中心挪。

这次的关键变化是：

- `FolderNode.build_cell_items_info(headers)` 现在是**模式感知**的
- 它会根据当前 `tree_columns` 只生成这个模式里真实存在的列
- 不再假设列名固定
- 备份 / 验证 / 动作库 三种模式都能分别对上

## 现在的处理方式

### 数据中心里
`FolderNode` 会根据传入的 `headers` 生成 `cell_items_info`，例如：

- `大小(本地)` 或 `大小`
- `有效文件夹` 或 `动作目录`
- `文件数` 或 `动作数`
- `资产列`
- `资产个人标签`
- `标签从规则文件`

### UI 构建 item 时
树项文本仍然按 UI 的方式设置，但在 item 构建完成后，会把当前模式的 `headers` 传给节点：

```python
headers = self.get_file_tree_headers_by_mode()
if hasattr(node, "build_cell_items_info"):
    node.cell_items_info = node.build_cell_items_info(headers)
```

这样就实现了你要的：

- 参考 UI 创建时怎么构建 item 文本
- 把这套逻辑迁移到数据中心
- 并且按模式自动适配列标题差异

---

## 你特别提醒的“不同模式列标题不一致”
这个已经按模式处理了，不会再写死某一套标题。

也就是说：

- `backup_file / verify` 模式会用它们自己的列头
- `action_library` 会用自己的 `动作目录 / 资产列 / 资产个人标签 / 标签从规则文件`
- 没有的列就不生成，不会乱塞
